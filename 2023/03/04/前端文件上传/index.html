    前端文件上传协议

    在一个文件上传请求中，通常包含以下内容：

    1. 请求方法：通常是POST方法，因为文件上传请求需要将文件内容作为请求体发送到服务器。
    2. 请求URL：上传文件的接口地址，可以是相对路径或绝对路径。
    3. 请求头部：包含请求相关的元信息，如User-Agent、Content-Type、Content-Length等，其中Content-Type的值通常是multipart/form-data，表示这是一个带有文件内容的表单请求。
    4. 请求体：请求体中包含要上传的文件内容，以及其他表单数据。文件数据通常是二进制流的形式，需要将其编码为multipart/form-data格式。对于非文件数据，也需要使用multipart/form-data格式编码，每个表单字段都需要有一个唯一的名字。

    例如，一个简单的文件上传请求可能如下所示：

    POST /upload HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/89.0.4389.82 Safari/537.36
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="file"; filename="example.txt"
Content-Type: text/plain

This is an example file.
------WebKitFormBoundary7MA4YWxkTrZu0gW
Content-Disposition: form-data; name="username"

john.doe
------WebKitFormBoundary7MA4YWxkTrZu0gW--

    其中，请求方法为POST，请求URL为/upload，请求头部中包含User-Agent、Content-Type等元信息，请求体中包含一个名为file的文件字段和一个名为username的非文件字段。在请求体中，使用了multipart/form-data格式来编码文件数据和非文件数据，使用boundary参数来指定不同字段之间的分隔符。每个字段包含了Content-Disposition和Content-Type两个参数，用于描述字段的名称、类型和内容。
